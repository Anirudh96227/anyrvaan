This is ten short films, and each one rebuilds an old computer exactly as it looked on screen — the Commodore 64, MS-DOS, Windows 95, the first iPhone, and six more in between. Here's the part that matters: none of it is a screenshot or an old recording. Every pixel was drawn again from scratch, to match what these machines really looked like when they were new.

The reason to do it that way comes down to a strange fact: you don't actually remember old computers as well as you think. You remember the *idea* of Windows 95 — the teal background, the Start button. You've forgotten the little grey banner running down the side of the Start menu, the exact way the buttons looked raised off the screen, the red counter in Minesweeper. Build it from memory and you'll make something that feels almost right and is wrong in a dozen tiny ways. And almost-right is worse than obviously fake, because anyone who was actually there feels that something's off without being able to say what.

So I chased the small stuff that most people never noticed but would miss without knowing it. The Commodore 64 starts up saying it has exactly 38,911 bytes free, then sits there loading from tape for an uncomfortably long time — because it truly did, and speeding it up to be "watchable" would give the whole game away. The old 1997 web page has a visitor counter that actually ticks up for you, a spinning UNDER CONSTRUCTION sign, a little tune playing. Get enough of these right and the memory snaps back into focus. There's real warmth in that moment of recognition — oh, *that's* what it looked like — and it only comes from getting the details honest.

The films are also strung together on purpose. Each one opens like a title card and ends on a blinking light that hands straight off to the next machine, so twenty-five years of computers play as one continuous run rather than ten separate clips. The care that went into looking things up instead of trusting memory is the subject of [I rebuilt old computers and learned my memory was lying](/blog/ui-archaeology/), and the wider collection of screen looks this pulls from is [a field guide to app animations](/work/ui-animation-atlas/).
